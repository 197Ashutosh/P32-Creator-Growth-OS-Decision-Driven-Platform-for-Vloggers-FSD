## 📌 Overview
**Creator Growth OS** is a sophisticated, decision-driven full-stack platform designed to transform raw YouTube performance data into actionable growth strategies. Developed as a specialized tool for vloggers, it utilizes a **Decoupled Cloud Architecture** to move beyond traditional analytics and provide a strategic roadmap for content success.

---

## 🎯 Problem Statement
Most creators suffer from **"Data Paralysis"**—they have access to YouTube Studio analytics but lack the logic to translate those numbers into their next upload. This gap leads to inconsistent posting, burnout, and stagnant growth.

---

## 💡 The "Decision-Driven" Solution
This system acts as a **Digital Mentor**, bridging the gap between raw data and human action by providing:
* **Real-time API Syncing:** Instant identity and metric validation via YouTube Data API v3.
* **LLM-Powered Strategy:** Custom growth targets generated by **Llama 3.1** via Groq Cloud.
* **Consistency Monitoring:** Database-backed streak tracking to build creator discipline.
* **Actionable Next-Steps:** Direct, two-sentence advice focused on immediate growth.

---

## 🏗️ System Architecture
The platform is built on a **Decoupled Architecture**, ensuring the UI and the Logic Engine are independent and scalable.

<p align="center">
  <img src="https://img.shields.io/badge/Architecture-Decoupled_Full_Stack-white?style=flat-square" alt="Arch">
</p>

* **Frontend:** Glassmorphism UI (HTML5, CSS3, JavaScript ES6) hosted on **Netlify**.
* **Backend:** Dockerized Java Application (OpenJDK 17) hosted on **Render**.
* **Intelligence:** Llama 3.1-8B-Instant LLM.
* **Persistence Layer:** MySQL Cloud (Aiven) for historical performance logging.

---

## 🚀 Execution Flow (The Demo Path)
1.  **Initialize:** Handshake with YouTube API via Channel ID to sync profile metadata.
2.  **Process:** Log performance metrics (Category, Views, Engagement) to MySQL via JDBC.
3.  **Analyze:** Process data through the Llama 3.1 LLM based on current vs. historical stats.
4.  **Deploy:** Output a "Next-Action Target" on the Glassmorphism dashboard.

## ⚙️ Deployment & Setup

### Backend (Dockerized Java)
The backend is containerized to ensure the Java Runtime Environment (JRE) is consistent across all cloud instances.
```dockerfile
# Build & Start Configuration
FROM eclipse-temurin:17-jdk
WORKDIR /app
COPY . /app
RUN javac -cp .:mysql-connector-j-9.6.0.jar CreatorGrowthOS.java
CMD ["java", "-cp", ".:mysql-connector-j-9.6.0.jar", "CreatorGrowthOS"]
```
Frontend Configuration
The frontend communicates with the decoupled backend via the following endpoint:
